Code Reading Club — Music Session

Making Music with Code

Strudel lets you write code that makes sound. You type a pattern, hit play, and it loops forever. Change the code while it's playing and the music updates instantly. No music experience needed — no install either, it runs in your browser.

open strudel.cc
shortcuts
Ctrl+Enter (⌘+↵ on Mac) play / update
Ctrl+. (⌘+. on Mac) stop
concepts — paste each snippet into strudel.cc and listen before reading on
first
Everything loops
Whatever you write plays on repeat in one fixed-length loop. More sounds crammed in = each one plays faster. Fewer sounds = more breathing room between them. Try it — paste this, play it, then add more words to the string and play again. 
sound("bd hh sd hh")
// add more words here — hear it speed up
drum sounds
What those short names mean
The short words inside sound("…") are drum sounds. Here's what each one actually sounds like:
bddeep thud — the lowest hit, the one you feel in your chest
sdsharp crack — a snappy hit, cuts through everything
hhshort tick — crisp and fast, keeps steady time
ohopen shhh — longer, breathy version of hh
cpclap — sharp hand-clap sound
mtmid tom — rounder, lower thud than bd 
// paste this and listen to each sound
sound("bd sd hh oh cp mt")
Try: change the order. Put bd first and last. Put two hhs in a row. Hear how the rhythm changes just by rearranging.
the most important thing
Silence — ~
A tilde ~ is an empty beat — a pause. It doesn't make a sound, it just takes up space in the loop. This is what separates music from noise. Listen to the difference: 
sound("bd sd bd sd")    // no gaps — relentless
sound("bd ~ sd ~")      // gaps on beats 2 and 4 — suddenly grooves
Try: move the ~ to different positions. Notice how the same sounds feel completely different depending on where the silence falls.
repeat
Multiply with *
Put * after any sound to play it multiple times within that one slot. hh*4 squeezes four ticks into the space one would take. 
sound("hh*2")
sound("hh*4")
sound("hh*8")  // very busy
volume
Turn it up/down — .gain()
.gain() sets how loud a line is. 1 is full volume, 0 is silent, anything in between is somewhere between those two. 
sound("hh*8").gain(0.2)  // quiet
sound("hh*8").gain(0.8)  // loud
melody
Playing notes — note()
So far everything has been rhythm — sounds with no pitch. note() lets you play actual pitches, like piano keys. Each note has a letter name (c d e f g a b) and a number for how high or low (higher number = higher pitch). A sequence of notes one after another is a melody. 
// paste this — it plays four notes in a row
note("c3 e3 g3 e3").sound("piano")

// try swapping piano for: casio  gm_violin  triangle  sawtooth
Try: change the notes. Swap e3 for eb3 — that one flat makes it sound darker. Change a 3 to a 4 to jump an octave higher.
the big one
Play multiple things at once — $:
Start any line with $: and it runs at the same time as every other $: line — each one looping independently. This is how you build something that has rhythm, bass, and melody all happening together. 
$: sound("bd ~ sd ~")                          // rhythm
$: sound("hh*8").gain(0.3)                     // quiet ticks
$: note("c3 ~ e3 ~ g3 ~ e3 ~").sound("piano")  // melody
Try: delete just one of the $: lines and play it. Notice how much the overall feel changes even though two thirds of the code is still there.
notes you can use
Every note is a letter plus a number. The letter is the pitch, the number is the octave (how high or low). 3 is a comfortable middle range — 4 is one step higher, 2 is lower. 
// the main letters (think piano white keys)
c3  d3  e3  f3  g3  a3  b3
c4  d4  e4  f4  g4  a4  b4   // same notes, one step higher

// flat notes — b means "flat" (slightly lower)
eb3  ab3  bb3

// these 5 notes sound good together — safe to mix freely
c3  eb3  f3  g3  bb3
